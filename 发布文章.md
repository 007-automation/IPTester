# 工控人必备：我写了个免费的网络诊断工具，支持设备识别+IP冲突检测

## 为什么写这个工具

我是个电气工程师，日常跟PLC、机器人、触摸屏打交道。

现场调试最烦的是什么？**不知道设备IP在哪。**

- 新厂房几十台设备，网段里到底有哪些在线？
- 谁设了重复IP搞到通讯断断续续？
- 这个IP到底是PLC还是HMI？

每次都打开cmd敲 `ping`，再敲 `arp -a`，一个IP一个IP试，烦死了。

上网一搜，Angry IP Scanner 是Java的难部署，Advanced IP Scanner 功能太通用不够工控。干脆自己写一个。

## 它能干什么

### 1. 一键扫全局域网

```
点「🔍局域网」→ 自动检测当前子网 → 点「🚀扫描」
3秒扫完 254 个IP，在线设备全列出来
```

### 2. 自动识别设备厂商

内置 160+ MAC OUI 数据库，扫描时自动匹配：

- `00:1C:06` → **Siemens SIMATIC**
- `00:00:BC` → **Rockwell / Allen-Bradley**
- `00:01:53` → **Omron**
- `00:0A:9F` → **Siemens**
- `00:60:25` → **ABB**
- `B8:27:EB` → **Raspberry Pi**

不用打开柜门看铭牌就知道谁是谁。

### 3. 设备类型推测

根据开放的 TCP 端口自动推测设备角色：

| 端口组合 | 推测结果 |
|---------|---------|
| 102 | 🔌 S7comm PLC（西门子） |
| 502 | 🏭 Modbus TCP 设备 |
| 502 + 80 | 🌐 Modbus Web 网关 |
| 44818 | 🔧 EtherNet/IP 设备 |
| 44818 + 2222 | 🇯🇵 Omron PLC (FINS) |
| 4840 | 📡 OPC UA 服务器 |
| 34964 | 🔌 Profinet 设备 |

### 4. IP 冲突检测

扫描完成后自动查 ARP 表。如果一个 IP 对应多个 MAC 地址——**IP 冲突！**

整行变橙红，弹窗告警，精确到冲突的 MAC 地址。

这个功能在现场排查通讯故障时救命——有时候设备通讯时断时续，查半天才发现是IP冲突。

### 5. 快照对比

```
第一天：扫描 → 保存快照
一个月后：扫描 → 加载快照对比
→ 自动标出：➕新增  ➖离线  🔀端口变更
```

定期巡检神器。谁偷偷改了端口、谁私下接了新设备，一对比就知道。

## 技术实现

- **语言**: C# 
- **框架**: .NET 8.0 Windows Forms
- **依赖**: 零！纯原生，不依赖任何第三方库

核心原理：
- Ping: `System.Net.NetworkInformation.Ping`
- 端口扫描: `System.Net.Sockets.TcpClient`，SemaphoreSlim 控制并发30线
- MAC 厂商: 内置字典，匹配 OUI 前6位
- ARP 冲突: 解析 `arp -a` 命令输出
- 设备推测: 端口组合模式匹配
- 快照: JSON 序列化/反序列化

编译出来就一个 `.exe`，U盘拷到任何 Windows 电脑上直接跑。
