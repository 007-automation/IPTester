# 🔍 IP Test Tool · 007

> 工控工程师的网络诊断利器 — Ping · 端口扫描 · 设备识别 · IP冲突检测

[![.NET](https://img.shields.io/badge/.NET-8.0-blue)](https://dotnet.microsoft.com/)
[![Platform](https://img.shields.io/badge/platform-Windows-lightgrey)]()

## ✨ 功能

| 功能 | 说明 |
|------|------|
| **Ping 连通性检测** | 单IP / CIDR子网 / IP范围批量扫描 |
| **TCP 端口扫描** | 30线程并发，4个预设（常用/Web/工控/全量）+ 自定义 |
| **🔍 局域网一键扫描** | 自动检测当前子网，一键扫全部 |
| **MAC 厂商识别** | 内置 160+ OUI，覆盖西门子/罗克韦尔/ABB/KUKA/发那科等 |
| **主机名解析** | 反向 DNS 查询 |
| **设备类型推测** | 根据端口组合智能推测：Modbus网关 · OPC UA服务器 · Omron PLC · Profinet设备 |
| **⚠️ IP 冲突检测** | ARP 表分析，同 IP 多 MAC 自动报警 |
| **📸 快照对比** | 保存扫描结果（JSON），下次加载自动对比变更 |
| **📋 导出** | CSV 格式导出 |

## 🚀 快速开始

### 方式一：直接运行（推荐）

1. 下载 `IPTester.exe`（Release 页）
2. 双击运行

### 方式二：从源码编译

```bash
# 安装 .NET 8.0 SDK
# https://dotnet.microsoft.com/download/dotnet/8.0

git clone https://github.com/007-automation/IPTester.git
cd IPTester
dotnet build -c Release
# 输出: bin/Release/net8.0-windows/IPTester.exe
```

### 方式三：Visual Studio

打开 `IPTester.csproj`，F5 运行。

## 📖 典型场景

### 1. 新厂房设备盘点
```
点「🔍局域网」→ 选工控预设 → 点「🚀扫描」
→ 几分钟扫完整个网段，在线设备、厂商、类型一目了然
```

### 2. 排查IP冲突
```
扫描完自动检查ARP表 → 同IP多MAC标红弹出警告
→ 精确定位冲突设备
```

### 3. 设备变更监控
```
第一天扫完 → 📸保存快照
一个月后再扫 → 🔄加载快照对比
→ 自动标出新增/离线/端口变更
```

### 4. 调试PLC通信
```
输入PLC网段 → 选工控预设 → 扫描
→ 看到设备类型、开放端口，快速定位通信问题
```
